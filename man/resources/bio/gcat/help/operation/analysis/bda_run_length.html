Calculates the run length for a given binary dichotomic partition. A run is a continuous sequence of tupels (codons) that belong to a partition which is embedded by runs of tupels that do not belong to that partition or vice versa. The output is 1) the minimum run length, 2) the average run length and 3) the maximum run length for all partitions. Currently, only codons (i.e. tuples of length 3) are allowed.

Parameter
The BDA set as defined in the BDA Tool is taken.

Example for Rumer partitions (0, 1):
Input sequence:AUU AAC GCA GCC GCU UCU UAC CAG UUG ACA UCA AAC UAA AGA CGG UAU GAG GAG ACC UGG ACC CAA AUG CGA GCC CGC GGG AUC CAC GAU CUU ACG CGG GUA ACC CGU AGA ACC UCA AGG
Analysis result:
0: min = 1, avg = 2.63, max = 6
1: min = 1, avg = 2.11, max = 3
